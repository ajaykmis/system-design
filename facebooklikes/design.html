# Regional CRDT Counter Merge class GlobalAggregator: def merge_regional_counts(self, post_id): us_count = get_regional_count('us-east', post_id) eu_count = get_regional_count('eu-west', post_id) asia_count = get_regional_count('asia-pacific', post_id) # CRDT merge - conflict-free return us_count.merge(eu_count).merge(asia_count)

Global Scale Features:

• Regional independence (low latency)
• Approximate counting for viral content
• HyperLogLog for unique user counts
• Cross-region eventual consistency
• Automatic failover between regions

🎯 System Design Interview Key Points

Design Decisions to Highlight:

1. Consistency Trade-off: Choose AP over C (eventual consistency OK for likes)
2. Partitioning Strategy: Shard by post_id for even distribution
3. Approximation: Use sampling for viral posts (>100K likes)
4. Regional Architecture: Process locally, sync globally
5. Failure Handling: Circuit breakers, fallbacks, monitoring
# Approximate counting for viral posts def increment_like_count(post_id, current_count): if current_count < 1000: return 1 # Exact counting elif current_count < 100000: return 10 if random() < 0.1 else 0 # Sample 10% else: return 100 if random() < 0.01 else 0 # Sample 1%
